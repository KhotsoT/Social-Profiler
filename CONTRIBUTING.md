# Contributing to Social Profiler

Thank you for your interest in contributing to Social Profiler! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers and help them learn
- Focus on constructive feedback
- Respect different viewpoints and experiences

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/social-profiler.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Follow the setup guide in `SETUP.md`
5. Make your changes
6. Test your changes
7. Submit a pull request

## Development Guidelines

### Code Style

- Use TypeScript for all new code
- Follow existing code patterns
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

### Commit Messages

Use clear, descriptive commit messages:

```
feat: Add Instagram API integration
fix: Resolve follower deduplication edge case
docs: Update API documentation
refactor: Simplify influencer categorization logic
test: Add tests for follower matching
```

### Pull Request Process

1. Update documentation if needed
2. Add tests for new features
3. Ensure all tests pass
4. Update CHANGELOG.md
5. Request review from maintainers

## Project Structure

- `backend/` - Express API server
- `frontend/` - Next.js web application
- `database/` - Database schemas and migrations
- `docs/` - Documentation
- `services/` - Microservices (future)

## Testing

- Write tests for new features
- Ensure existing tests pass
- Aim for good test coverage

## Documentation

- Update relevant docs when adding features
- Keep API documentation current
- Add examples for new features

## Questions?

Open an issue for questions or discussions about contributions.
